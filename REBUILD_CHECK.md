# リビルド確認手順

## 目的
UcOperationInternal.cs の修正が正しくビルドされているか確認します。

## 手順

### 1. Visual Studio でソリューションをクリーン

```
メニューバー → ビルド → ソリューションのクリーン
```

### 2. 出力ウィンドウを開く

```
メニューバー → 表示 → 出力
または Ctrl + Alt + O
```

### 3. ソリューションをリビルド

```
メニューバー → ビルド → ソリューションのリビルド
```

### 4. ビルド結果を確認

出力ウィンドウで以下を確認してください：

#### ✅ 成功の場合:
```
========== ビルド: 成功 1、失敗 0、最新の状態 0、スキップ 0 ==========
```

#### ❌ 失敗の場合:
```
========== ビルド: 成功 0、失敗 1、最新の状態 0、スキップ 0 ==========
```

エラー一覧ウィンドウ（表示 → エラー一覧）で具体的なエラーを確認してください。

### 5. エラーがある場合

以下の情報を error.md.txt に保存してください：
1. 出力ウィンドウの内容
2. エラー一覧の内容

---

## よくあるビルドエラー

### エラー: debugLogCounter が宣言されていない

これは、前回の修正が正しく保存されていない可能性があります。

**解決方法**: UcOperationInternal.cs を再度編集して保存してください。

### エラー: 構文エラー

コードの一部が正しく保存されていない可能性があります。

---

## ビルドが成功したら

次の手順に進んでください：
1. F5 でデバッグ実行
2. ブロックプログラミングエンジンを選択
3. FormScript タブでプログラムを確認/設定
4. Operation タブで Start をクリック
5. 出力ウィンドウのログを確認

エラーがあれば、error.md.txt に保存して共有してください。
