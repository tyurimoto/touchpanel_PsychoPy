# テスト手順 (Test Instructions)

## 目的
Startボタンが正しく動作しているか確認します。

## 手順

### 1. アプリケーションを起動
- Visual Studio で F5 を押してデバッグ実行
- 出力ウィンドウを開く (Ctrl + Alt + O)

### 2. 起動ログを確認
出力ウィンドウに以下が表示されるはずです：
```
=================================================
[BUILD VERIFICATION] FormMain_Load - ビルド日時: XXXX-XX-XX XX:XX:XX
=================================================
デバッグモード（完全ダミー）で起動しました
```

**もしこのログが表示されない場合、リビルドが必要です。**

### 3. Operation タブを開く
- 画面上部の "Operation" タブをクリック
- **エラーが出ずに Operation 画面が表示されることを確認**

**もしここでクラッシュする場合は、error.md.txt にログを保存してください。**

### 4. Start ボタンを探す
Operation タブ内に **"Start"** ボタンがあるはずです。

以下を教えてください：
- [ ] Start ボタンは表示されていますか？
- [ ] Start ボタンは有効ですか（グレーアウトしていませんか）？
- [ ] Start ボタンの位置はどこですか？（スクリーンショットがあれば最適）

### 5. Start ボタンをクリック
**実際に Start ボタンをクリックしてください。**

### 6. 結果を確認

#### A) MessageBox が表示された場合
✅ **ボタンハンドラーは動作しています！**

MessageBox を閉じた後、出力ウィンドウで以下のログを確認してください：
```
=== START BUTTON CLICKED ===
Current State: Idle
EnableDebugMode: True
[Idle] Start command received. EnableDebugMode=True
```

**これらのログが表示されたら、error.md.txt に全ログを保存してください。**

#### B) MessageBox が表示されない場合
❌ **ボタンハンドラーが実行されていません**

以下を確認してください：
1. 本当に "Start" ボタンをクリックしましたか？
2. 別のボタンではありませんか？
3. ブロックプログラミングモードで特別なボタンがありますか？

### 7. error.md.txt にログを保存
**上記の結果に関わらず、出力ウィンドウの内容を error.md.txt に保存してください。**

---

## 報告してください

以下の情報を教えてください：

1. **Step 3**: Operation タブを開いたときにクラッシュしましたか？
2. **Step 4**: Start ボタンは表示されていますか？
3. **Step 5**: Start ボタンをクリックしましたか？
4. **Step 6**: MessageBox は表示されましたか？
5. **Step 7**: error.md.txt の内容

これにより、問題の原因を特定できます。
